But AdventureWorks doesn't have a 'Profit' column.

You'll calculate it during your ETL.

For example:
Profit = LineTotal - (OrderQty × StandardCost)


So your mapping would look like this:

| Fact Column | Source     |
| ----------- | ---------- |
| Profit      | Calculated |

## DimCustomer

| Source Table   | Source Column        | Used? | Gold Table  | Gold Column  | Transformation                                  | Quality Check          |
| -------------- | -------------------- | :---: | ----------- | ------------ | ----------------------------------------------- | ---------------------- |
| Sales.Customer | CustomerID           |   ✅   | DimCustomer | CustomerID   | None                                            | NULL, Duplicate        |
| Sales.Customer | PersonID             |   ✅   | DimCustomer | Join         | Join to Person.Person                           | Orphan                 |
| Sales.Customer | StoreID              |   ✅   | DimCustomer | CustomerType | CASE (StoreID IS NULL = Individual, ELSE Store) | None                   |
| Sales.Customer | TerritoryID          |   ✅   | FactSales   | TerritoryKey | Lookup to DimTerritory                          | Orphan                 |
| Person.Person  | BusinessEntityID     |   ✅   | DimCustomer | Join         | Join with PersonID                              | Duplicate              |
| Person.Person  | FirstName            |   ✅   | DimCustomer | FirstName    | TRIM                                            | NULL (optional), Blank |
| Person.Person  | LastName             |   ✅   | DimCustomer | LastName     | TRIM                                            | NULL (optional), Blank |
| Person.Person  | FirstName + LastName |   ✅   | DimCustomer | FullName     | Concatenate                                     | None                   |
